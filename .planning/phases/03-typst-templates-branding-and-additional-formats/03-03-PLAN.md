**typst-show.typ** — Rewrite to support format variant routing. The key challenge: Pandoc template syntax does not support conditional `#import` or conditional function calls in the way a programming language does. However, we CAN use the Pandoc `$if()$` conditional to select which function to call.

Approach: Import ALL three format functions at the top, then use `$if(typstR.format-variant)$` conditionals to call the right one. Since only ONE `#show:` call can exist, use a wrapper approach:

```
// Import all format overlays
#import "formats/workingpaper.typ": working-paper
#import "formats/article.typ": article
#import "formats/brief.typ": policy-brief

// Default to working-paper if no format-variant specified
$if(typstR)$
$if(typstR.format-variant)$
$if(typstR.format-variant/match:"article")$
#show: article.with(
  // ... all field wiring (same as current working-paper.with block) ...
)
$elseif(typstR.format-variant/match:"brief")$
#show: policy-brief.with(
  // ... all field wiring ...
)
$else$
#show: working-paper.with(
  // ... all field wiring ...
)
$endif$
$else$
#show: working-paper.with(
  // ... all field wiring ...
)
$endif$
$else$
#show: working-paper.with(
  // ... all field wiring (minimal, no typstR fields) ...
)
$endif$
```
