Create the `inst/quarto/extensions/typstR/templates/` directory and five shared module files by extracting logic from the monolithic typst-template.typ:

**base.typ** — Exports `apply-base-styles(primary-font, title-font, margins, accent-color, body)` function that:
- Calls `set page(paper: "us-letter", margin: margins, numbering: "1")` where margins defaults to `(x: 1in, y: 1in)`
- Calls `set text(font: primary-font, size: 11pt)` where primary-font defaults to `"Linux Libertine"`
- Calls `set par(justify: true, leading: 0.65em)`
- Calls `set heading(numbering: "1.1")`
- If accent-color is not none, applies `show heading: set text(fill: accent-color)`
- Renders body as last expression
- All style parameters have sensible defaults so callers can omit them

**titleblock.typ** — Exports `render-title-block(title, subtitle, date, report-number)` function that:
- If title is not none: `align(center)[#text(size: 16pt, weight: "bold")[#title]]` + `v(0.5em)`
- If subtitle is not none: `align(center)[#text(size: 13pt)[#subtitle]]` + `v(0.3em)`
- If date is not none: `align(center)[#text(size: 10pt)[#date]]` + `v(0.3em)`
- If report-number is not none: `align(center)[#text(size: 10pt, style: "italic")[#report-number]]` + `v(0.5em)`
- All parameters default to `none`

**authors.typ** — Exports `render-author-block(authors, affiliations, anonymized)` function that:
- If anonymized is true, returns immediately (renders nothing)
- Builds affiliation ID-to-index mapping: iterate `affiliations` with `.enumerate()`, insert each `aff.at("id", default: str(i))` mapped to `i + 1` into a dictionary
- Renders authors centered with superscript affiliation numbers: for each author, get name via `.at("name", default: "")`, collect affiliation IDs from `.at("affiliations", default: ())`, look up indices, render as `[#name#super(sups.join(","))]`. If author has `.at("corresponding", default: false) == true`, append `*` after superscripts
- Renders affiliation list below: `v(0.3em)` then each affiliation as `#super(str(i + 1))#aff.at("name", default: "")`, joined by `[; ]`
- Renders corresponding author email below affiliations: find the author with `corresponding == true`, get email via `.at("email", default: "")`, unescape with `.replace("\\@", "@")`, render as `v(0.3em)` + `align(center)[#text(size: 9pt)[Corresponding author: #email-display]]`
- ORCID display: for each author with `.at("orcid", default: none) != none`, include ORCID after name or as a separate line (use discretion for clean layout)

**abstract.typ** — Exports `render-abstract(abstract, keywords, jel, abstract-label)` function that:
- abstract-label defaults to `"Abstract"` (brief format will pass `"Summary"`)
- If abstract is not none: `pad(x: 1.5cm)[*#abstract-label.* #abstract]` + `v(0.5em)`
- If keywords array is not empty: `pad(x: 1.5cm)[*Keywords:* #keywords.join(", ")]` + `v(0.3em)`
- If jel array is not empty: `pad(x: 1.5cm)[*JEL Classification:* #jel.join(", ")]` + `v(0.3em)`
- `v(1em)` at the end

**appendix.typ** — Exports `start-appendix()` function that:
- Calls `set heading(numbering: "A.1", supplement: [Appendix])`
- Calls `counter(heading).update(0)`
- This follows the verified Typst forum pattern

All module functions use the `.at('field', default: value)` pattern for safe dictionary access (established in Phase 2). No `@preview` imports allowed.
